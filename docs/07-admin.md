# 07 · ADMIN — 운영자 화면 명세

> SaaS 운영에 필수인 관리자 영역. 사용자 영역(`/dashboard` 등)과 **완전히 분리된 라우트**(`/admin`)와 권한으로 보호한다.
> 운영자가 회원·템플릿·영상·비용을 보지 못하면 서비스를 운영할 수 없다 → 선택이 아니라 필수 인프라.

---

## 1. 권한 구조 (확정: 2단계 역할)

`users/{userId}.role` 필드로 구분. 기본값 `"user"`.

| 역할 | 값 | 권한 |
|---|---|---|
| 일반 사용자 | `user` | 본인 프로젝트만 |
| **운영 스태프** | `staff` | 어드민 **읽기** 전체 + **제한적 수정**(아래 표 참조) |
| **슈퍼관리자** | `superadmin` | 모든 읽기/쓰기/삭제 + 역할 부여 + 시스템 설정 |

### 액션별 권한 매트릭스
| 액션 | staff | superadmin |
|---|---|---|
| 회원 목록·상세 조회 | ✅ | ✅ |
| 회원 크레딧 조정 | ❌ | ✅ |
| 회원 정지/삭제 | ❌ | ✅ |
| 역할(role) 변경 | ❌ | ✅ |
| 템플릿(Style Pack) 조회 | ✅ | ✅ |
| 템플릿 활성/비활성 토글 | ✅ | ✅ |
| 템플릿 생성/편집/삭제 | ❌ | ✅ |
| 생성 영상 조회·재생 | ✅ | ✅ |
| 영상 강제 삭제(신고/위반) | ✅ | ✅ |
| 비용·매출 대시보드 | ✅ | ✅ |
| 시스템 설정(API키 정책 등) | ❌ | ✅ |

> 보안: 권한 검사는 **클라이언트 UI 숨김만으로 끝내지 말 것.** Vercel API 라우트와 Firestore 보안 규칙에서 서버사이드로 `role` 재검증. Firebase Auth Custom Claims에 role을 심어 규칙에서 검사하는 방식 권장(`request.auth.token.role`).

## 2. 어드민 화면 구성 (`/admin`)

```
/admin
  /            대시보드 (핵심 지표 요약)
  /members     회원 관리
  /templates   템플릿(Style Pack) 관리
  /videos      생성 영상 관리
  /voices      나레이션 보이스 관리
  /billing     비용·매출
  /settings    시스템 설정 (superadmin 전용)
```

### 2.1 대시보드 (`/admin`)
- 오늘/이번 달 지표: 신규 가입, 생성 영상 수, 총 외부 API 원가(costLog 합산), 매출, 진행 중 렌더 작업 수.
- 에러 알림: 실패한 renderJobs, 이미지 생성 실패율(한글 오타 재생성 포함).

### 2.2 회원 관리 (`/admin/members`)
- 목록: 이메일, 가입일, 플랜, 잔여 크레딧, 누적 사용 원가, 생성 영상 수, 상태.
- 검색·필터(플랜/상태/가입일), 페이지네이션.
- 상세: 해당 회원의 프로젝트·영상·크레딧 거래 내역.
- 액션(superadmin): 크레딧 가감, 정지/해제, 역할 변경, 삭제.

### 2.3 템플릿 관리 (`/admin/templates`) — **이 시스템의 핵심 운영 화면**
사용자분이 짚은 대로 **3가지 책임**이 여기서 갈린다:

1. **(어드민) 템플릿 정의** — Style Pack을 생성/편집. `03-data-schema.md`의 Style Pack 스키마를 폼으로 편집:
   - 이름·설명, 이미지 프롬프트 템플릿/네거티브, `textStrategy`(in-image/overlay/hybrid), 폰트, 오버레이 에셋(한지/낙관 업로드), 4대 Planner 기본값, 팔레트, 사용자 슬라이더 범위.
   - 활성/비활성 토글(staff 가능): 비활성 팩은 사용자 화풍 선택에서 숨김.
2. **(어드민→유저) 썸네일 등록** — 각 팩의 미리보기 샘플 프레임(1~3장)을 업로드/지정. 이게 사용자 화풍 선택 화면(`05-design-system.md` 4절 ③)에 노출되는 그 이미지.
   - 권장: 팩으로 실제 생성한 대표 장면을 썸네일로 등록(립서비스가 아니라 실제 출력 품질을 보여줌).
3. **버전·노출 순서** — 팩 정렬 순서, 신규/베타 배지.

> 데이터: Style Pack을 코드 상수가 아니라 **Firestore `stylePacks/{packId}` 컬렉션**으로 승격(어드민이 코드 배포 없이 편집 가능하게). 단 프롬프트 템플릿 같은 민감/핵심 값은 superadmin만 편집.

### 2.4 생성 영상 관리 (`/admin/videos`)
- 전체 생성 영상 목록: 제작자, 제목, 화풍, 길이, 생성일, 원가, 상태.
- 재생(서명 URL), 검색·필터.
- 신고/위반 영상 강제 삭제(staff 가능) + 사유 기록.
- 모더레이션 큐(선택): 공개 갤러리를 운영할 경우 승인 대기열.

### 2.6 비용·매출 (`/admin/billing`)
- costLog 집계: 항목별(이미지/TTS/LLM/렌더) 원가 추이, 영상당 평균 원가, 플랜별 마진.
- 크레딧 발행/소진, 결제 내역.
- 이상 탐지: 원가 급증 사용자(어뷰징) 플래그.

### 2.7 시스템 설정 (`/admin/settings`, superadmin)
- 기본 이미지 품질(high/medium), 재생성 최대 횟수, 동시 렌더 한도.
- 보이스 제공자 토글(OpenAI/ElevenLabs), 기본 LLM 제공자.
- 신규 가입 허용/차단, 점검 모드.

## 3. 나레이션 보이스 + 미리듣기 (사용자 영역 + 어드민 영역 공통)

> 사용자분 지적대로 누락됐던 부분. 보이스 선택 전 **미리듣기 필수.**

### 3.1 제공자별 미리듣기 방식 (검증됨, 2026 기준)

**ElevenLabs** — 미리듣기가 쉬움:
- List voices 엔드포인트로 사용 가능한 보이스 목록을 검색·필터·페이지네이션으로 조회 가능.
- **각 보이스 메타데이터에 preview URL이 포함**되어 있음 → 우리는 그 URL을 `<audio>`로 재생만 하면 끝(추가 합성 비용 0).
- 보이스마다 언어·억양·성별·나이 라벨 제공 → 필터 UI에 활용.

**OpenAI TTS** — 미리듣기 URL을 따로 제공하지 않음:
- 패리티 방법: **짧은 고정 샘플 문장**(예: "안녕하세요, 오늘의 이야기를 시작합니다.")을 보이스별로 1회 합성 → Storage에 캐싱 → 이후 그 캐시 오디오를 미리듣기로 재생. (보이스 9~13종 × 언어 몇 개 = 일회성 소량 비용)

### 3.2 데이터 모델 (`voices/{voiceId}` 컬렉션)
```jsonc
{
  "id": "vo_warm_f_ko",
  "provider": "elevenlabs" | "openai",
  "providerVoiceId": "...",        // 제공자 측 실제 ID
  "displayName": "따뜻한 여성 (한국어)",
  "language": "ko",
  "gender": "female",
  "tags": ["warm", "narration"],
  "previewUrl": "https://.../preview.mp3",  // EL: 제공 URL / OpenAI: 캐싱한 샘플
  "enabled": true,
  "tier": "free" | "premium"        // ElevenLabs 프리미엄은 상위 플랜 전용
}
```

### 3.3 사용자 화면 동작
- 보이스 선택 카드 각각에 ▶ 미리듣기 버튼 → `previewUrl` 재생.
- 언어·성별·톤 필터.
- `tier: premium` 보이스는 무료 플랜에서 잠금 표시(업셀 포인트).

### 3.4 어드민 화면 동작 (`/admin/voices`)
- 보이스 풀 관리: 노출/숨김 토글, 표시명·태그 편집, tier 지정.
- ElevenLabs: List voices 동기화 버튼(신규 보이스 가져오기 + previewUrl 갱신).
- OpenAI: 샘플 문장 재합성 버튼(미리듣기 캐시 갱신).

## 4. 페이즈 반영
- **Phase 0**: `users.role` 필드 + Custom Claims 토대 + `/admin` 라우트 보호(빈 골격).
- **Phase 1**: 보이스 데이터 모델 + 사용자 미리듣기(보이스 선택이 Phase 1 입력 화면에 있으므로 같이).
- **Phase 3~4**: 어드민 회원/영상/비용 화면(운영 시작에 맞춰).
- **Phase 2~4**: 템플릿 관리 화면(Style Pack을 Firestore로 승격하는 시점).
- (상세 순서는 `06-phases.md` 갱신본 참조)
