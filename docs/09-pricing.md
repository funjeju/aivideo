# 09 — 유료화 / 요금제 설계

> 상태: **설계 확정, 구현 전** (과금 토글 `settings.billingEnabled` 기본 OFF).
> 기존 인프라: `checkBillingGate`(승인 시 잔액확인) + 후차감(생성 후 costLog) + 면제(billingExempt). 이걸 정찰제·선차감으로 고도화한다.

## 1. 환율 (고정)
- **1 크레딧 = 10원**.
- 충전팩: 5,000원=500 / 10,000원=1,100(+10%) / 30,000원=3,600(+20%).

## 2. 정찰 단가 (마진 ≈3배, "보여준 값 = 차감 값")
| 항목 | 단가 |
|---|---|
| 생성 low | 12 크레딧/장면 |
| 생성 medium | 30 크레딧/장면 |
| 생성 high | 80 크레딧/장면 |
| 원고 수정 | 2 크레딧 |
| 이미지 재생성 | 12 크레딧 |
| 재렌더(내용 동일) | 무료 |
- 환산: **1분(9장면) low = 108크레딧(1,080원) / 2분(17장면) low = 204크레딧(2,040원)**.
- 마진 손잡이 = "3배"(원가 변동·실패·무료티어 흡수). 숫자만 바꾸면 전체 조정.

## 3. 요금제 (티어 + 기능 매트릭스)
| 기능 | 무료 | Lite (9,900/월) | **Pro (29,000/월)** |
|---|---|---|---|
| **동시 제작(멀티큐)** | 1편 | 1편 | **3~5편** |
| 우선 처리(큐 우선) | – | – | **✓** |
| 포함 크레딧 | 체험 1분×2편 | 1,200/월 | 4,000/월 |
| 장면 수정 | ✗ | ✓ | ✓ |
| 워터마크 | 있음 | 제거 | 제거 |
- 충전 크레딧은 모든 등급 병행 사용. 포함 크레딧 소진 시 충전분 차감.

## 4. 동시 제작(멀티큐) = Pro 전용 락
- 원가 거의 0(병렬은 총 compute 동일, 대기 0)인데 체감 가치 큼 → **고마진 업셀**. 무료·Lite의 직렬(1편) 대기 = 전환 압력.
- **UI**: 별도 메뉴 없음. 대시보드에서 여러 프로젝트가 동시에 진행 중으로 보이는 것 자체가 기능. 무료·Lite가 1편 처리 중 새로 시작 시 → "Pro로 업그레이드" 모달.
- **어뷰즈 차단(중요·서버측 강제)**: user 문서에 `activeCount` **원자 카운터**(Firestore 트랜잭션). 작업 시작 +1 / 완료·실패·취소 -1. 진입 게이트(승인·생성·렌더 enqueue)에서 `activeCount >= 플랜한도`면 API가 거절(409/402). → **새 탭·창으로 우회 불가**(UI가 아니라 서버에서 막음). 트랜잭션이라 동시 클릭에도 카운트 안 꼬임.
- 시스템 상한: 렌더 워커 maxScale=6(전체). "Pro N편"은 그 유저가 점유 가능한 슬롯 수이며, 6 초과는 큐 대기. 필요 시 maxScale만 상향.

## 5. 과금 정합성
- **선차감(hold) + 트랜잭션**: 시작 시 예상 크레딧을 미리 잡음, 실패 시 환불(증분). (현재 후차감은 동시 실행 시 잔액 초과 가능 → 교체)
- 차감 자체는 `FieldValue.increment`(원자적)이라 이중차감·누락은 이미 없음. 문제는 게이트가 racy + 후불 → 선차감으로 해결.
- **거래내역 ledger** 컬렉션: 충전/차감/환불 기록(정합·분쟁 대응).

## 6. 무료 티어
- 가입 시 **1분 영상 2편** 체험. **수정 불가**. **워터마크**(우하단, 둘 다 일관). 유료 전환 시 워터마크 제거.

## 7. 결제(충전/구독)
- 한국 PG: 포트원(PortOne)/토스페이먼츠/카카오페이. 충전팩 + 구독(Lite/Pro) webhook → 크레딧 증분(트랜잭션).

## 8. 구현 페이즈
1. **가격 엔진(`pricing.ts`) + 예상 표시 UI + 선차감/환불 + ledger** — PG 없이 내부 크레딧만으로 동작.
2. **수정 과금** — 장면 편집 버튼에 단가 표시 + 게이트/차감.
3. **무료 티어 + 워터마크** — 신규 2편 무료·수정잠금, 워터마크 합성(워커).
4. **결제 연동** — PG 충전팩 + 구독.
5. **동시 한도/멀티큐(Pro) + 우선순위** — `activeCount` 카운터, 플랜별 한도, (옵션)고우선 큐.
6. **관리자/정산** — 매출·차감 리포트, 환불.
