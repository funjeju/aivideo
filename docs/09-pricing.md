# 09 — 원가 분석 & 요금제 정책

> 상태: **설계 확정, 구현 전**(과금 토글 `settings.billingEnabled` OFF).
> 가격 원칙: **원가 × 2.5배**. 크레딧 단위 **1 크레딧 = 10원** 고정. 환율 가정 ₩1,380/$.

---

> **화질 정책: 전 영상 low 고정**(`/api/images`에서 강제). medium/high는 비용·예측가능성 때문에 제품에서 미사용. 티어 차이는 **크레딧 양 · 멀티큐(동시제작) · 길이 · 워터마크**로만 둔다.

## 1. 원가 분석 (2분 영상 = 17장면 · 나레이션 ~884자 · low 고정)

장면 수 = 길이÷7초. 2분 ≈ 17장면 기준 외부 API 실측 원가(low):

| 단계 | 모델/서비스 | 원가(low) |
|---|---|---|
| 원고(LLM) | gpt-4o 1회 | $0.03 |
| 음성 TTS | OpenAI tts-1 / Google Chirp3-HD¹ | $0.02 |
| Vision 분석 | gpt-4o × 17장면 | $0.13 |
| **이미지 생성** | gpt-image-2 × 17장면 | **$0.34** |
| 렌더+저장 | Cloud Run + Storage | $0.02 |
| **원가 합계** | | **≈$0.54** |
| **원화 환산** | | **≈745원** |

¹ Google Chirp3-HD는 월 100만자 무료 → 우리 규모(편당 ~884자)에선 사실상 $0. 보수적으로 $0.02 잡음.

**핵심**: 원가의 ~60%는 **이미지 생성** + Vision. 둘 다 장면 수에 비례 → 원가는 **길이(장면 수)에만** 선형으로 좌우됨.

## 2. 크레딧 정찰 단가 (1 크레딧 = 10원, ×2.5)

**"보여준 값 = 차감 값"** — 토큰 변동은 마진으로 흡수, 사용자에겐 항상 정확한 숫자.

| 항목 | 단가(크레딧) | 환산 |
|---|---|---|
| 생성 (low 고정) | **10 / 장면** | 100원 |
| 원고 수정 | 2 / 장면 | 20원 |
| 이미지 재생성 | 10 / 장면(생성가 동일) | 100원 |
| 재렌더(내용 동일) | **무료** | 0 |

환산 예: 1분 = 9×10 = **90크레딧(900원)** · 2분 = **170(1,700원)** · 5분 = **430(4,300원)** · 10분 = **860(8,600원)**.

### 충전팩
| 금액 | 크레딧 | 보너스 |
|---|---|---|
| 5,000원 | 500 | – |
| 10,000원 | 1,100 | +10% |
| 30,000원 | 3,600 | +20% |

## 4. 핵심 정책 (우리가 합의한 것 — 구현 시 반영)

- **동시 제작(멀티큐) = Pro부터**: 원가 거의 0(병렬은 총 compute 동일)인데 체감 가치 큼 → 고마진 업셀. 무료·Lite는 동시 1편(직렬) → 전환 압력.
  - 어뷰즈 차단: **서버측 `activeCount` 원자 카운터**(Firestore 트랜잭션). 시작 +1/종료 -1, 진입 게이트에서 한도 초과면 거절 → **새 탭/창 우회 불가**(UI 아닌 서버 강제).
- **선차감(hold) + 트랜잭션**: 시작 시 예상 크레딧 선점, 실패 시 환불. (후차감은 동시 실행 시 잔액 초과 가능 → 교체). 차감은 `FieldValue.increment`로 원자적. **거래내역(ledger)** 컬렉션 보관.
- **무료 워터마크**: 무료 영상 우하단 워터마크(브랜드 노출=우리 이득), 유료 전환 시 제거. (아웃트로는 전 영상 공통)
- **결제**: 한국 PG(포트원/토스/카카오페이) — 충전팩 + 구독 webhook → 크레딧 증분.

## 5. 구현 페이즈
1. 가격 엔진(`pricing.ts`, 단일 출처) + 예상 표시 UI + 선차감/환불 + ledger (PG 없이 내부 크레딧만)
2. 수정 과금(장면 편집 버튼에 단가)
3. 무료 티어 + 워터마크(워커 합성)
4. 결제 연동(PG 충전팩 + 구독)
5. 동시 한도/멀티큐(activeCount, Pro+) + 우선순위 큐
6. 관리자/정산(매출·차감 리포트, 환불)
