# 05 · DESIGN SYSTEM — 디자인 토큰 · shadcn/ui · MCP

> 결정: **미니멀 화이트 골격(참고 시안 1번 "DrawNarrate" 컨셉) + 동양적 포인트(C안 절충)**.
> 콘텐츠(흰 종이/한지 위 그림)와 UI가 같은 "종이" 위에 있는 통일감이 핵심 근거.

---

## 1. 디자인 방향 결정과 근거

- **베이스**: 미니멀 화이트. 우리 대시보드는 밝은 종이색 썸네일로 가득 차므로, 라이트 UI 위에서 콘텐츠가 자연스럽게 스며든다. (다크 UI에서는 흰 썸네일이 형광등처럼 떠서 시끄러워짐 → 전체 다크 테마 채택 안 함)
- **포인트**: 순백 대신 옅은 한지 톤 웜 화이트 + 먹색 차콜 + 단청 액센트 1색.
- **예외 1 — 프리뷰 스테이지만 다크**: 영상 편집 화면의 캔버스 주변 영역만 차콜 배경(Figma/프리미어가 캔버스 주변을 어둡게 두는 원리). 흰 캔버스 영상의 경계를 살리기 위함. **전체 다크 테마가 아니라 한 컴포넌트의 배경 처리.**
- **예외 2 — 손그림 모티프는 비기능 영역만**: empty state·온보딩·로딩 화면에만 손그림 일러스트. UI 골격에는 쓰지 않음(교수 세그먼트 신뢰감 보호).

## 2. 디자인 토큰 (CSS 변수 / Tailwind 확장)

```css
:root {
  /* 종이 / 배경 */
  --paper:        #FAF8F4;   /* 웜 화이트 (앱 기본 배경) */
  --paper-raised: #FFFFFF;   /* 카드/패널 */
  --paper-sunken: #F2EEE7;   /* 입력 필드/구분 영역 */

  /* 먹색 텍스트 */
  --ink:          #2A2A2E;   /* 본문/제목 */
  --ink-soft:     #5A5A60;   /* 보조 텍스트 */
  --ink-faint:    #9A9A A2;  /* 캡션/비활성 */

  /* 단청 액센트 (기본값: 적색 계열. 사용자 확정 대기) */
  --accent:       #C73E3A;   /* 단청 적색 — 주 액센트 */
  --accent-soft:  #E8C9C7;   /* 액센트 배경/호버 */
  /* 대안 액센트(청록): #2E7D6B / soft #C5E0D8 — 사용자 선택 시 교체 */

  /* 경계/그림자 */
  --line:         #E5E0D8;
  --shadow-sm:    0 1px 2px rgba(42,42,46,.06);
  --shadow-md:    0 4px 16px rgba(42,42,46,.08);

  /* 프리뷰 스테이지 (편집 화면 캔버스 주변 한정) */
  --stage-bg:     #232327;
  --stage-line:   #3A3A40;

  /* 반경 */
  --radius:       12px;
}
```

> **액센트 컬러는 사용자 최종 확정 대기.** 기본값 단청 적색(#C73E3A). 청록(#2E7D6B) 대안 제공. Claude Code는 토큰만 바꾸면 전역 교체되도록 변수로만 참조할 것(하드코딩 색상 금지).

## 4. 핵심 화면별 디자인 의도

1. **대시보드** (`/dashboard`): 프로젝트 썸네일 그리드. 흰 종이 카드 위 그림. 상단에 "새 프로젝트" 단청 액센트 버튼. 진행 중 렌더는 진행률 바.
2. **생성 입력** (`/create`): 큰 입력창(주제) 또는 드롭존(문서). 그 아래 **3개 선택**(길이/화풍/보이스)을 큰 탭/카드로. 저마찰 = 화면 하나에 끝.
3. **화풍 선택**: 여기서만 각 Style Pack을 화려하게 미리보기(실제 샘플 프레임). UI 자체는 조용하지만 이 카드만 콘텐츠로 빛남.
4. **원고 승인** (`/project/[id]`): 장면 카드 리스트(narration + visualIntent). 인라인 편집. 하단 "승인하고 영상 만들기" 액센트 버튼.
5. **진행률**: Worker 진행률 실시간 바 + 단계 라벨(원고/이미지/렌더링).
6. **편집/프리뷰**: **캔버스 주변만 --stage-bg 다크.** 우측 장면 리스트, 컷별 "이 그림 다시 / 이 문장 수정". (참고 시안 2번 InkMotion 구조를 이 화면에만 적용)

## 5. shadcn/ui + MCP 연결 (Claude Code 행동 지침)

UI 컴포넌트는 **shadcn/ui를 MCP로 연결**해 설치·조회한다.

### 시작 시 Claude Code가 할 일
1. **shadcn MCP 서버 연결 여부 확인.**
   - 연결됨 → MCP를 통해 컴포넌트 목록 조회 후 필요한 것 설치.
   - **미연결 → 임의로 CLI 추측 설치하지 말고, 사용자에게 "shadcn/ui MCP를 연결해 달라"고 요청.** (이 명세의 README 체크리스트 항목)
2. 컴포넌트 설치 후, 위 디자인 토큰(2절)을 `globals.css`/Tailwind 설정에 주입해 shadcn 기본 테마를 우리 토큰으로 덮어쓴다.
3. shadcn의 기본 라운드/색을 그대로 쓰지 말고 `--radius`, `--accent`, `--paper` 등 우리 토큰에 매핑.

### 예상 사용 컴포넌트 (MCP로 설치)
`button`, `card`, `tabs`, `dialog`, `dropdown-menu`, `input`, `textarea`, `progress`, `skeleton`, `toast(sonner)`, `tooltip`, `avatar`, `badge`, `slider`(화풍 미세조절), `sheet`(편집 패널), `scroll-area`.

> Claude Code: 위 목록은 가이드일 뿐, 실제 필요 시 MCP로 조회해 최신 컴포넌트명을 확인하고 설치. 버전·이름 추측 금지.

## 6. 반응형 / 접근성
- 모바일: 대시보드·입력·승인 화면은 1열 스택. 편집/프리뷰는 데스크톱 우선(모바일은 보기 전용).
- 대비: 먹색/종이 조합으로 WCAG AA 충족. 액센트는 텍스트보다 액션에.

## 6.5 다크모드 (토대는 지금, 완성은 Phase 5)

> 방침: **"나중에 손댈 수 없을 만큼 공수가 커지지만 않으면 된다"** → 토대만 지금 깔고 완성은 미룬다.

- **지금(Phase 0) — 공수 거의 0**: 모든 색을 CSS 변수로만 참조(이미 그렇게 설계됨). `.dark` 변수 세트를 미리 정의(아래). `themePref` 필드(`03-data-schema.md`)와 헤더 토글 UI 배치. 단 Phase 1~4 기본값은 light + "프리뷰 스테이지만 다크", 전체 다크 토글은 베타 표시.
- **나중(Phase 5)**: 핵심 난점은 **흰 종이/한지 썸네일이 다크 배경에서 형광등처럼 뜨는 문제**(디자인 결정의 근거 그 자체). 단순 색 반전으로 안 되고 썸네일 카드에 어두운 매트/테두리를 따로 처리해야 함 → 이 작업만 Phase 5로.

```css
.dark {
  --paper:        #1B1B1F;
  --paper-raised: #232327;
  --paper-sunken: #18181B;
  --ink:          #ECEAE4;
  --ink-soft:     #B0AEA8;
  --ink-faint:    #6A6A70;
  --accent:       #E0625E;   /* 다크 배경용으로 약간 밝게 */
  --accent-soft:  #4A2A29;
  --line:         #34343A;
  /* 프리뷰 스테이지는 라이트/다크 공통으로 항상 어둡게 유지 */
}
```

## 6.6 다국어 (i18n) — 두 레이어 분리 (토대는 지금)

언어가 두 층으로 흐른다. 혼동 금지:

1. **UI 언어 (`uiLocale`)**: 버튼·메뉴·안내문. `next-intl`(또는 동등) 으로 **문자열을 코드에서 분리**. 지금(Phase 0): 분리 구조 + `uiLocale` 필드 + 토글만, 실제 번역은 Phase 5. 미루면 고통스러운 항목 → 하드코딩 한글 문자열 금지, 전부 메시지 키로.
2. **콘텐츠 언어 (`contentLocale`)**: 생성 영상의 나레이션·자막·in-image 텍스트 언어. 차별화 핵심(벤치마크 50개 언어급). 파이프라인 3곳에 언어 변수: ①LLM 원고 ②TTS 보이스 ③in-image 텍스트. **언어별 textStrategy 분기 필요** — 영어는 in-image가 잘 되지만 한글·일본어·아랍어는 오타율 제각각 → Style Pack에 `textStrategyByLocale` 오버라이드 여지. 지금은 필드 자리만, 실제 튜닝은 Phase 2 한글 실측 이후.

## 7. 손그림 모티프 가이드 (비기능 영역 한정)
- empty state: "아직 프로젝트가 없어요" + 붓으로 그린 듯한 빈 캔버스 일러스트.
- 온보딩: 3컷 손그림 튜토리얼.
- 로딩: 붓이 선을 긋는 마이크로 애니메이션(우리 제품 정체성 = '그리는 도구').
