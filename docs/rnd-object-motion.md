# R&D 브리프 — 객체 분리 기반 "살아있는 모션"

> 상태: **별도 R&D 프로젝트로 분리**(2026-06-18 발의). 본 서비스(easyshorts)와 독립 실험.
> 한 줄: 밑그림 처리 시 **개체를 세분화해 분리**해두면, "따라 그리기(드로잉 리빌)"를 넘어 **분리된 조각을 움직여 자연스러운 모션**을 줄 수 있다.

---

## 0. 목표
드로잉 리빌이 끝난 (혹은 진행 중인) 그림에 **은은한 움직임**을 얹어 정적 이미지를 "살아있게" 만든다. 컷아웃 애니메이션·Live2D·시차(parallax)와 같은 원리. 차별화 포인트: "그려지고 → 살아 움직이는" 설명 영상.

## 1. 핵심 아이디어
- 장면 이미지를 **객체 단위(또는 더 잘게)로 세그멘테이션 분리**
- 각 조각에 **시간별 transform(이동·스케일·회전·워프)** 을 얹어 모션 생성
- 프레임별 렌더(node-canvas)라 **프레임마다 조각 위치를 바꾸면** 모션이 됨

## 2. 난이도 3단계 (낮은 것부터)

### 🟢 1단계 — 객체별 미세 모션 / 시차 (고ROI, 현 구조로 가능)
- 이미 분리된 객체에 **부유(둥실)·미세 줌/드리프트·전경·배경 시차(parallax)**
- 정적 그림 → "살아있는" 느낌. **체감 품질 대비 가성비 최고**(스크라이브·Golpo류가 쓰는 기법)
- 재설계 불필요 — 객체 mask에 시간별 transform만 추가

### 🟡 2단계 — 2.5D 깊이 시차
- 전경(인물)/중경/배경을 **깊이 레이어로 분리** → 다른 속도로 이동 → 입체감
- 세그멘테이션 한 단계 더(깊이/전경 분리) 필요

### 🔴 3단계 — 캐릭터 파츠 리깅 (끝판, 큰 R&D)
- "최대한 세분화 → 자연스러운 모션"의 종착점: 팔·머리 등 **파츠 단위 움직임**
- 필요: **의미적 파츠 인식**(이게 팔/머리/몸통) + **리그(뼈대)** + **메시 워프**(Live2D식)
- 평면 이미지를 퍼펫으로 만드는 작업 → 난이도·리스크 높음

## 3. 현재 토대 (재활용 가능한 자산)
- 렌더 엔진이 **이미 객체 단위로 분리**해서 리빌·영역 채움을 함 (`renderCore.ts`의 골격/획/영역 로직)
- **node-canvas 프레임별 렌더** → 프레임마다 객체 transform을 줄 수 있는 구조 (raw RGBA → ffmpeg 파이프)
- 즉 🟢 1단계는 "그린 뒤 객체를 움직이는 레이어 추가" 수준으로 접근 가능

## 4. 기술 진입점
- `src/lib/render/renderCore.ts` + `worker/src/render-engine/renderCore.ts` (**두 곳 동기화 필수**)
- 객체 분리/영역 로직(이진화→세선화→획·영역). 모션은 **객체별 transform을 시간축에 추가**
- 변경 시 **sceneHash 버전 증가**(세그먼트 캐시 무효화) + **워커 재배포** 필요
- 더 잘게 분리하려면: 별도 세그멘테이션(SAM류 segment-anything, 또는 connected-components 강화)

## 5. 리스크 / 트레이드오프
- **세분화 과하면 분리 오류** → 조각이 따로 놀아 어색. **적당한 단위**가 더 자연스러움.
- 모든 모션 = **렌더엔진 작업 + 워커 재배포**(이미지 생성 단계 아님). 본 서비스 렌더 안정성에 영향 가능 → **분리 실험이 안전.**
- 3단계(리깅)는 별도 모델·파이프라인이라 사실상 신규 프로젝트.

## 6. 추천 진행 순서
1. **🟢 미세 모션/시차 프로토타입** — 한 장면에 객체 부유+배경 시차부터. 체감 확인.
2. 좋으면 → 2.5D 깊이 분리
3. 🔴 파츠 리깅은 별도 R&D 트랙(장기)

## 7. 분리 이유
본 서비스는 "드로잉 리빌"이 안정·런칭 단계. 모션은 **리스크 높은 고도화**라 본 렌더 파이프라인과 섞으면 안정성 위협. **독립 실험 프로젝트**에서 검증 후, 성공한 단계만 본 엔진에 역이식.
