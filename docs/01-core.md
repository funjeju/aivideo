# 01 · CORE — 프로젝트 철학과 목표

## 1. 한 문장 정의

> **주제 또는 자료를 입력하고 몇 번의 선택만 하면, 원하는 그림체(화이트보드·수묵담채·민화 등)의 지식 전달용 드로잉-리빌 영상을, 편당 수천 원 이하의 원가로 자동 생성하는 교육·교양 특화 영상 SaaS.**

벤치마크는 Golpo AI (whiteboard explainer 영상 생성기). 우리는 그 검증된 구조를 토대로 하되, **그들이 갖지 못한 "동양적 화풍 다양성"이라는 축**으로 차별화한다.

---

## 2. 우리가 싸우지 않는 것 / 싸우는 것

- **싸우지 않음**: Sora·Veo·Kling 같은 범용 시네마틱 영상 생성기. (프레임 단위 디퓨전, 고비용, 일관성 붕괴 문제)
- **싸움**: "지식을 명확히 전달하는 설명 영상" 수직 시장. 핵심 가치는 **명확성·일관성·저비용·속도**이지 사실적 영상미가 아니다.

### 왜 이 구조가 저비용인가 (설계의 근거)
영상을 프레임 단위로 생성하지 않는다. AI가 생성하는 것은 ①원고(LLM) ②장면 이미지(장면당 1~2장) ③나레이션(TTS) ④연출 계획(JSON)뿐이다. **실제 영상은 렌더링 엔진(CPU)이 정적 이미지 위에 마스크 리빌 애니메이션을 입혀 만든다.** 따라서 GPU 비용이 디퓨전 대비 자릿수가 다르게 낮다.

> 참고 원가 (2026년 6월 기준, 외부 API 직접 호출 추정치 — 변동 가능):
> - 50초 숏폼: 약 $1.5~2 (이미지 GPT Image 2 high 약 8장이 지배적)
> - 5분 롱폼: 약 $5.5~7 (이미지 약 30장)
> - 이미지가 전체 원가의 80~90%. → 비용 최적화의 핵심 레버는 이미지 생성 전략.

## 3. 핵심 통찰 — "일관성은 모델 문제가 아니라 설계 문제다"

화풍 일관성 붕괴를 걱정할 필요가 적은 이유:

1. **화면 골격(제목·라벨·화살표·도형·차트)은 정보 구조이며 그림체와 무관하게 동일한 문법을 따른다.** Style Pack은 이 골격 위에 입히는 그림체일 뿐이다.
2. **AI 생성 이미지는 장면당 단독 모티프 1~2개**(산, 매화, 저울, 뇌 등)에 국한된다. 컷마다 다른 소재이므로 "앞 컷과 동일해야 한다"는 요구 자체가 거의 없다. 필요한 것은 동일성이 아니라 **톤 유지**.
3. 한지 질감·낙관·여백 등은 **렌더러의 고정 오버레이 레이어**로 처리 → 모든 프레임이 같은 "종이" 위에 있는 통일감을 거의 공짜로 확보.

> **예외**: 동일 캐릭터가 영상 내내 반복 등장하는 서사형 콘텐츠(예: 호랑이가 주인공인 이야기)만 캐릭터 시트 1회 생성 후 레퍼런스 재사용 모드가 필요. 정보·교양형 콘텐츠에는 불필요.

### 텍스트 렌더링에 대한 현재 가설 (검증 대상)
관찰된 레퍼런스 영상에서 텍스트는 **이미지 안에 함께 생성**된 것으로 보인다(곡면을 따라 휘는 라벨, 그림 선과 동일한 잉크 질감). 2025~26 세대 이미지 모델(GPT Image 2)부터 가능해진 능력이다.
- **1순위 기술 검증 과제**: 한글 in-image 텍스트의 오타율 실측. 영어와 달리 한글은 자모 조합 복잡도로 오타율이 높을 수 있음 (확실하지 않음 — 반드시 파일럿 측정).
- **폴백 전략**: 한글 오타율이 허용선을 넘으면, 곡면/사물 통합 텍스트는 이미지 생성에 맡기고 **평평한 제목·라벨은 붓글씨 폰트 오버레이**로 처리하는 하이브리드. 오버레이 텍스트는 오타 0% + 수정 무료라는 이점.

## 4. 타겟 세그먼트

| | 주력: 지식 크리에이터 | 부차: 교수·교육자 |
|---|---|---|
| 목적 | 채널 업로드용 콘텐츠 대량 생산 | 강의 보조 자료 |
| 기본 입력 모드 | 생성 모드(주제→LLM 작성) | 충실 모드(자료 업로드) |
| 길이 | 숏폼 50초~3분 | 롱폼 5~15분, 챕터 분할 |
| 화풍 니즈 | 채널 아이덴티티 = 화풍 차별화 | 깔끔·중립 우선, 동양풍은 인문/역사 과목 매력 포인트 |
| 과금 | 월 구독 + 크레딧 | 단건 결제 / 기관 라이선스 가능성 |

> **중요**: 위 표는 "기능 분리"가 아니라 "기본값 추천 차이"다. 두 세그먼트는 **동일 파이프라인을 공유**하고, 입력 모드와 과금만 다르다. 교수용 충실 모드가 크리에이터의 품질 보증 경로도 겸한다.

---

## 5. 입력 모드 2종 (제품의 핵심 분기)

### 생성 모드 (Generate)
- 입력: 주제 프롬프트 (예: "한계효용 체감의 법칙을 3분 영상으로")
- LLM이 원고를 처음부터 작성.
- 자유도 높음. **환각(사실 오류) 리스크 존재** → 원고 승인 체크포인트 필수.

### 충실 모드 (Faithful)
- 입력: 원고/강의자료 업로드 (PDF, DOCX, TXT, MD)
- LLM은 **원본의 사실·논리·용어·전개 순서를 100% 준수**.
- 단, 형식은 전면 변환: ①문어체→나레이션 구어체 ②목표 길이 맞춤 축약/확장 ③장면 분할 ④장면별 시각화 지시문 작성.
- 즉 **"내용 동결, 형식 변환"**. UX에서 "내가 쓴 내용 그대로(O), 내 문장 그대로(X)"임을 명확히 안내.
- 옵션: "문장도 그대로 읽기" 모드 → LLM은 장면 분할 + 시각화 지시만 수행 (비용 더 낮음, 사용자 녹음 나레이션 모드와 연결).

### 공통 체크포인트: 원고 승인 (Script Approval)
- 두 모드 모두 **이미지 생성(=진짜 비용 발생) 직전에 원고 텍스트를 사용자에게 보여주고 승인받는다.**
- 이 단계에서의 수정은 비용이 거의 0. 영상 완성 후 사실 오류 발견 시 재생성 비용을 원천 차단.
- "선택 3번으로 끝" 원칙에 1클릭(승인)이 추가되지만, 지식 콘텐츠 특성상 마찰이 아니라 안심 장치로 작용.

## 6. 사용자 플로우 (저마찰 원칙)

```
[입력] 주제 입력 OR 문서 업로드
   ↓
[선택 3개] ① 길이(50초 / 3분 / 10분)  ② 화풍(Style Pack)  ③ 목소리 톤
   ↓
[원고 생성] LLM이 원고 + 장면 분할 초안 작성
   ↓
[원고 승인] ← 사용자 확인/수정 (비용 0 체크포인트)
   ↓
[자동 생성] 이미지 → 연출 JSON → 렌더링 (진행률 표시)
   ↓
[결과] 영상 + 컷 단위 사후 편집("이 그림 다시" / "이 문장 수정")
```

시안 고르기 단계 없음. 원샷 생성 후 컷 단위 사후 수정 모델.

## 7. Style Pack 라인업 (초기 3종, 확장 가능 구조)

1. **클래식 화이트보드** — Golpo 패리티. 시장 검증된 기본값. 교수 세그먼트 디폴트.
2. **수묵담채** — 차별화의 핵심. 한지 텍스처 + 먹선 + 담채 포인트색 + 붓글씨 한글 타이틀. 심리·철학·역사 콘텐츠와 최적합.
3. **민화/조선** — 호랑이·까치·책가도 모티프, 오방색 포인트. 한국사·문화 콘텐츠용, 썸네일 차별화 효과 큼.

> 웹툰풍·우키요에 등은 후속 확장. 각 팩 내부에 색온도·여백 밀도 미세 조절 슬라이더 제공.
> 세 팩 모두 **그림체만 다르고 정보 구조 문법은 동일** → 연출 계층(Planner) 코드 재사용.

## 8. 용어집

- **Style Pack**: 그림체 1종의 완전한 정의 묶음. (`03-data-schema.md`에 스키마)
- **Scene Spec (장면 JSON)**: 영상의 단일 진실 소스. 렌더러 입력. (`03-data-schema.md`)
- **Reveal Planner**: 장면 내 객체 공개 순서 결정 (무엇을 먼저).
- **Stroke Planner**: 객체 내부 생성 순서 (어떻게 그려지는 것처럼).
- **Flow Planner**: 마스크 제거 방향 (시선 이동).
- **Sync Planner**: 나레이션-시각 동기화 리듬.
- **Render Worker**: Cloud Run 위의 영상 렌더링 전용 서버.
- **생성 모드 / 충실 모드**: 입력 방식 2종.
